Authentication Header (AH)

A protocol used to provide connectionless integrity and data origin authentication for IP packets.

Description

The Authentication Header (AH) is a component of the IPsec suite of protocols, designed to provide authentication and integrity for IP packets. It operates at the network layer, ensuring that the data payload of a packet has not been altered during transit and that it originates from a legitimate source. AH achieves this by appending an authentication header to the IP packet, which includes a cryptographic checksum calculated over the packet's contents. This checksum is generated using a shared secret key, allowing the recipient to verify the authenticity of the sender. AH does not provide confidentiality, meaning the data is not encrypted, but it secures the integrity and authenticity of the data. This is particularly useful in scenarios where data integrity is critical, such as in virtual private networks (VPNs) and secure communications over the internet. AH can be used in combination with the Encapsulating Security Payload (ESP) for a comprehensive security solution, as ESP provides both encryption and additional security features.

Examples

  • Used in Virtual Private Networks (VPNs) for secure communications.
  • Implemented in secure network protocols to ensure data integrity.

Additional Information

  • AH operates without encrypting the data, focusing solely on authentication and integrity.
  • It can work in both transport mode and tunnel mode within the IPsec framework.
